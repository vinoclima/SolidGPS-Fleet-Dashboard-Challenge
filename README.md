# SolidGPS — Fleet Dashboard Challenge

A single Python script that reads a CSV snapshot of GPS tracking devices and
produces a fully self-contained `fleet_dashboard.html` — no setup, no
dependencies, open in any browser.

---

## How to Run

```bash
python fleet_dashboard_generator.py
```

Place `fleet_status.csv` in the same folder as the script. That's it.

**Output:** `fleet_dashboard.html` — open it in any browser.

---

## What It Produces

### 1. Summary Cards
Total device count broken down by status — Active, Idle, Offline, Low Battery,
Maintenance. Colour-coded to match the map markers.

### 2. Interactive Map
Every device plotted at its GPS coordinates using Leaflet.js. Markers are
colour-coded by status. Click any marker to see device name, battery level,
last seen time, and location.

### 3. Device List
Full table of all valid devices showing status badge, battery bar, last seen
(human-readable — "2h ago", "3d ago"), and location.

### 4. Anomaly Report
Any dirty or unexpected records are flagged transparently at the bottom of the
page — not silently dropped. The fleet manager can see exactly what was wrong
and why.

---

## How Dirty Data Is Handled

The real CSV contained several problematic records. Here is how each was handled:

| Device | Issue | Action |
|--------|-------|--------|
| TRK031 | Missing name, battery, lat, lon | Skipped — cannot plot without coordinates |
| TRK033 | Battery reported as 150% | Clamped to 100%, flagged in anomaly report |
| TRK034 | Latitude value is `not_a_lat` | Skipped — invalid coordinate |
| TRK035 | Last seen timestamp is in 2027 | Flagged as future date, still displayed |

The decision was to **never crash silently**. Every anomaly is logged and shown
to the fleet manager so they can investigate the device or fix the data at
source.

## Technical Decisions

**Standard library only**
No pandas, folium, requests, or any third-party packages. Just `csv`, `json`,
`os`, `sys`, and `datetime`. Runs anywhere Python 3.6+ is installed.

**Self-contained HTML**
All device data is injected into the HTML as an inline JavaScript variable at
generation time. The only external call is Leaflet.js loaded from CDN — no
other files are needed.

**Anomaly handling over silent failure**
GPS devices in the field send bad data. The script validates every field
(coordinates within Australia's bounding box, battery clamped to 0–100,
timestamps checked for future dates) and surfaces issues rather than hiding
them.

**Readable code over clever code**
Every function has a docstring explaining what it does and why. Variable names
are descriptive. Logic is broken into clear steps so any developer can follow
it without needing comments to decode it.
